# Daily Report — 2026-06-24

**Project:** ShelfScape · **Author:** Ruiqi Huang
**Theme of the day:** turn the school project into a real, expandable codebase, and make scanning smart (auto rows + auto tiling).

## Summary

Took ShelfScape from a single-folder school submission to a proper Git project
with a Gradle build, then replaced the manual "Scan Region" workflow with an
automatic scan that finds shelf rows and splits each row into the fewest tiles
the model needs to read small print.

## What changed (3 commits, all pushed to `main`)

### 1. `afd5cb0` — Core ShelfScape (trim to the product)
- Created a standalone Git repo (was previously nested inside another repo) and
  pushed to `github.com/ShinanoMusashi/Shelf_Scan`.
- Removed everything that was school/scaffolding only: grading artifacts
  (TEST_MATRIX, design report), IDE files (`.idea`, `.iml`, `.project`,
  `.classpath`), the offline demo fallback (`DemoRecognizer` + sample image), and
  the `run`/`setup` scripts.
- Result: 22 source files, ~2,935 lines — just the app + MiniCPM-V integration.

### 2. `8fe2878` — Gradle build + standard layout
- Added the Gradle wrapper (8.14), `application` plugin (`mainClass =
  shelfscape.app.MainApp`), compiling for Java 17.
- Moved sources to the standard `src/main/java/` layout — what Gradle and a
  future Android module expect.
- Run with `./gradlew run`.

### 3. `edc3b65` — Smart scan: auto rows + adaptive tiling
- New UI-free package **`shelfscape.scan`** (pure geometry, reusable on any
  platform): `RowDetector`, `TilePlanner`, `DetectionMerge`.
- **`ScanController`** rewritten: a quick first pass finds the rows, then each row
  is re-scanned in just enough tiles (rows that read fine cost no extra scan);
  seam duplicates are merged.
- **`Book`** gained a `row` field (saved at the end of the line; old saves still
  load). **`ShelfPanel`** bookshelf view now draws multiple stacked rows.
- Net +485 / −94 lines across 7 files.

## Verification

- Compiles via `./gradlew compileJava` and plain `javac` (22 files).
- Scan logic tested headlessly: row clustering (2 rows, correctly ordered),
  tile counts (dense row → 3 tiles, sparse row → 1), tile overlap, and
  duplicate merging all behave as expected.

## Known / pending
